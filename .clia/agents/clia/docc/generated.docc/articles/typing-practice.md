Sharpen speed and accuracy with short, focused drills that mirror the way we write DocC and typed Swift CLI options. Keep sessions short, repeat often.

## Drills

- Warmup (2 minutes)
  - Lowercase alphabet, then uppercase; keep posture and rhythm.
  - Symbols we use a lot in docs: backticks, brackets, angle brackets.
- Snippets (3 minutes)
  - DocC fragments: `@Metadata { @TitleHeading("…") }`, `<doc:…>` links, fenced blocks.
  - CLI flags: `--long-option`, kebab-case identifiers, enum values.
- Patterns (5 minutes)
  - Copy small sections from the cheat sheet and style kit without peeking at the keyboard.
  - Aim for smooth lines over raw speed; reduce corrections.

## Targets

- Baseline: 55–65 WPM, < 2% error rate on mixed text
- Focus sets: 40–50 WPM when practicing symbols-heavy text (DocC/CLI)

## Verify

- Run a 5‑minute session and record WPM + errors.
- Paste one paragraph into a spell/grammar pass; aim for zero corrections.
